# 进展报告 — 肿瘤科笔记的结构化提取

**项目：** 用本地 LLM 从肿瘤科门诊笔记中做忠实的结构化信息提取（及面向患者的总结）。
**覆盖周期：** 开发至 2026-06。**数据：** UCSF CORAL（去标识化的乳腺 + 胰腺肿瘤科笔记；200 条未标注用于开发，40 条专家标注 held-out 用于评估）。

---

## 1. 目标

把自由文本的肿瘤科笔记里临床重要的事实抽取成一组固定字段（现用药、分期、转移、疗效、受体/分子状态、各类计划等），遵循四条不可妥协的原则（按优先级）：(1) 精确忠实、绝不幻觉；(2) 不遗漏重要内容；(3) 简单词汇；(4) 患者能看懂。更长期的目标是把结构化输出转成面向患者的总结信。

## 2. 我们做了什么（方法）

系统是一个**围绕冻结的开源模型构建的推理编排层（inference harness）——按设计不做任何模型微调**。底座模型是**本地 vLLM 部署的 Qwen2.5-32B-Instruct-AWQ**，权重从不修改。这是有意为之：数据全程留在本地（符合 HIPAA 关切）、不需要标注训练集、且完全可复现。所有适配都在推理期通过四层完成：

- **多阶段提取** —— 不用单 prompt，而是分阶段处理：6 个独立的 Phase-1 prompt（就诊原因、诊断、检验、检查发现、现用药、治疗变化），2 个带依赖的 Phase-2 prompt（治疗目标、疗效，注入 Phase-1 结果作为上下文），以及计划提取阶段（用药/手术/影像/检验/基因/转诊/随访）。拆分任务降低模型单次调用的认知负担。
- **5-gate 验证级联**（逐字段）：JSON 格式修复 → schema/key 校验 → 具体化+语义对齐 → 忠实度修剪（"有依据就保留，只有明确矛盾或捏造才清空"）→ 时态过滤（删除已完成/过去项）。
- **两部确定性词典**：肿瘤药物白名单（约 158 种药）+ 18,739 词的医学术语表（按 8 年级词汇过滤）。
- **约 135 个确定性后处理 hook 族**，编码通用临床规则（如区域 vs 远处淋巴结、把"暂停化疗"识别为一种计划变更、受体一致性等）。

## 3. 开发过程（"training"）

**没有做任何梯度训练。** 取而代之的是在 200 条未标注开发笔记上跑一个**评估驱动的迭代开发循环**：

1) 跑 pipeline；2) 逐字段对照原文人工审查（自然语言临床判断，不用脚本判定）；3) 缺陷分级（P0 幻觉 / P1 放错字段或方向错 / P2 小问题）；4) 通过改 prompt、加 gate、或写通用 hook 修复；5) 跑回归集（上一轮所有出问题样本 + 30% 随机无问题样本）。乳腺约 15 轮、胰腺约 18 轮。我们刻意优先写通用临床规则而非针对测试集硬编码，以便改进能迁移到未见过的笔记；当剩余问题变成"打地鼠"式长尾（且被 vLLM 跨运行的非确定性放大）时停止。

## 4. 医生评审结果

- **信件质量（临床反馈）：** 医生审阅了生成的患者信件，指出最激进的简化版本（V33）**改变了原意 / 过度简化**；结论是*准确性必须优先于简化*，更忠实的早期风格更可取。此前的乳腺/胰腺信件包评审中被标记的问题已全部修复。
- **三方部署对比（pipeline vs 裸 Qwen vs GPT-4o，40 封信）：** 裸 Qwen baseline **0/40 可发送**（45% 泄露了去标识符 `*****`，12.5% 幻觉）；GPT-4o 不符合 HIPAA（云端）且更长更密。**pipeline 达到零幻觉、零去标识泄露、约 97.5% 可发送、数据全本地** —— 即同一个开源模型裸用不可用，但放进 harness 后可临床部署，且在部署指标上超过云端 GPT-4o baseline。
- **held-out 提取准确性（乳腺 v31）：** P0 = 0，P1 = 0；受体状态 93%，治疗目标方向 100%。

## 5. 结果与分析 —— 提取消融实验（当前焦点）

我们用一个干净的受控对比来隔离 harness 的价值，在全部 40 个 held-out 样本上：**PL**（完整 pipeline）vs **BL**（**同一个**底座模型、**同一套**字段 schema，单 prompt、无后处理）。唯一变量 = harness。每个样本的每个字段都由经过临床 brief 的审查者评判（8 个 subagent，每个审 5 个样本，主审复核；只用自然语言阅读，不用脚本评分）。

在计入评分的题集上（618 个字段对比）：**PL 更好 131 · 打平 475 · BL 更好 12**。在两个系统真正不同的题里，PL 占绝大多数胜场，BL 基本不在高价值字段上获胜。最强、最站得住的区分是 **现用药"抗癌药 vs 非癌家用药"的识别（PL 胜，约 35:0）** —— baseline 几乎总把降压/降糖/止痛家用药倒进抗癌药字段，而 pipeline 能隔离出真正的治疗方案。其他 PL 占优的高价值字段：分期、远处转移、分子/遗传结果。那 12 个 BL "胜"全部**非幻觉**，都是次要的计划/摘要字段（影像/检验/操作子计划、化验摘要、支持用药）或边界措辞问题，不是核心诊断错误。

**解读：** 多数题打平，是因为两个系统共用同一个强底座模型；harness 可测量的价值集中在那些"裸 prompt 系统性出错"的高知识字段（药物分类、分期、转移细节、分子结果）。这正是我们想要的"大量打平 + 明确的 PL 胜 + 核心字段 BL 几乎零胜"的格局。

## 6. 为什么换方向，以及换成什么方向

**为什么不再以信件生成为主线：** 患者信件质量难以客观评估，而医生评审表明简化步骤会与忠实性此消彼长——而忠实性正是我们排在第一位的原则。信件正在变成瓶颈，也是系统中风险最高、最难度量的部分。

**新方向 —— 仅提取的消融实验：** 我们把重心移回结构化提取任务，因为它 (a) 是信件所依赖的基础，(b) 可由医生逐字段客观评估，(c) 是一个干净的受控实验（固定底座模型+schema，harness 作为唯一变量）。这把贡献重新表述为**一个让冻结的开源模型变得临床可靠的推理 harness**，用逐字段的 PL-vs-BL 对比来证明，而不是靠主观的信件点评。

我们也据医生意见收紧了评估本身：删掉了临床无评分价值的题（就诊类型、治疗目标方向、就诊原因总结），把受体题设为仅乳腺（胰腺没有 ER/PR/HER2），并在 prompt 中加入一条原则，推动模型**抽取具体事实而非推断出一个工整的总结**（并据此重跑了全部 40 个样本，修好了诸如把"chemo break 暂停化疗"误判成"无用药变化"的情况）。

## 7. 下一步

1. **医生盲评。** 已备好两个面向医生的打分页面：一个带标签版，一个**盲评 A/B 版**（两个系统只显示为 A/B，隐藏身份），让医生在无偏见下评 PL 更好 / 打平 / BL 更好。从一名或多名医生处收集量化胜率。
2. **泛化检查。** 审计约 135 个 hook 是否含针对测试集的逻辑；在新笔记 / 更多癌种上验证规则成立。
3. **锁定确定性。** 降低对 greedy 解码跨运行波动的依赖，使最终数字稳定。
4. **以准确性优先重做总结。** 提取锁定后，在明确"准确性优于简化"的约束下重新引入面向患者的输出。
5. **撰写论文。** 整合成一篇"用于冻结本地 LLM 上临床可靠提取的推理 harness"论文，以 PL-vs-BL 消融和部署指标对比作为核心结果。
